# D01 M1B feasibility estimate for dynamic prime-bias verification

Assumption: the verification needs dynamic x sweeps up to hundreds of trillions, not stored prime lists.
Method target: segmented counting plus residue tallies, not explicit enumeration of all primes.

## Prime counts by scale
| X | pi(X) ~ X/log X | 8-byte storage for all primes |
|---|---:|---:|
| 1e+12 | 3.619e+10 | 0.3 TB |
| 1e+13 | 3.341e+11 | 2.7 TB |
| 1e+14 | 3.102e+12 | 24.8 TB |
| 3e+14 | 9.000e+12 | 72.0 TB |

## Segmented-work sketch
| Segment size | Segments to 1e14 | Segments to 3e14 |
|---|---:|---:|
| 1e+07 | 10,000,000 | 30,000,000 |
| 1e+08 | 1,000,000 | 3,000,000 |

## Feasibility verdict
- Storing all primes is a non-starter at 1e14 and above.
- A segmented, checkpointed pipeline is feasible for a targeted modulus/residue set.
- A full dynamic-x sweep over many x-values is cluster-scale, not single-workstation scale.
- If Koyama wants a reproducible pilot, the right next step is a narrow modulus set and a bounded x grid.
