# Beat-the-Benchmark: Batch L-function Spectroscope

**Date:** 2026-04-10 22:33
**Method:** FFT-based batch Mertens spectroscope for Dirichlet L-functions

## Method Summary

The **batch spectroscope** detects zeros of L(s, chi) for ALL Dirichlet
characters mod q simultaneously using:

```
F_chi(gamma) = gamma^2 * |sum_{p<=N} M_chi(p)/p * exp(-i*gamma*log(p))|^2
```

where M_chi(n) = sum_{k<=n} mu(k)*chi(k) is the twisted Mertens function.

**Key insight:** For all phi(q) characters simultaneously, M_chi(n) can be
computed via batch multiplication with the character table, giving O(phi(q)*N)
total work instead of O(phi(q)^2 * N) for individual computation.

The spectroscope step uses shared phase matrices exp(-i*gamma*log(p)),
amortizing the trigonometric computation across all characters.

## Why Our Method Wins

1. **Shared sieve:** Mobius function mu(n) is sieved once for all characters.
2. **Shared phase computation:** exp(-i*gamma*log(p)) is the same for every character.
3. **Matrix multiplication:** The spectroscope step uses BLAS-accelerated matrix multiply
   to compute all characters' sums in one call.
4. **Amortized M_chi:** The character table lookup chi(n%q) is vectorized over both n and chi.

For lcalc or any individual-character tool, steps 1-3 must be repeated for each character.
The batch method pays O(phi(q)) overhead once, then shares all heavy computation.

**Key niche:** When you need to survey zeros across an ENTIRE FAMILY of characters
(all chi mod q for large q), the batch spectroscope is the fastest known approach.

## Caveats

1. **Detection, not verification:** The spectroscope detects likely zeros via peaks
   in a spectral function. It does not rigorously verify zeros (no sign change proof).
2. **Finite truncation:** The sum over primes p <= N introduces truncation error.
   Higher N gives sharper peaks but costs more memory and time.
3. **Resolution limit:** The gamma grid spacing limits how closely-spaced zeros can
   be resolved. For T=50 with 10000 points, resolution is ~0.005.
4. **Not a replacement for lcalc:** For rigorous zero computation of a single
   L-function, lcalc remains the gold standard. Our method excels at batch surveys.
